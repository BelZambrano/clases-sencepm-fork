Eventos jQuery

Sitio demo: eventos comunes con jQuery

Esta página está diseñada para una Clase 1–2: cada sección demuestra un evento distinto con un ejemplo simple. Abajo tienes un “Log” que va registrando todo lo que haces.

Consejo pedagógico: prueba cada sección y mira el Log. Eso te ayuda a entender “qué evento se dispara” y “cuándo”.

Evento: click

Ocurre cuando haces clic en un elemento (botón, div, imagen, etc.). Es el evento más usado para ejecutar acciones.

Eventos: mouseenter y mouseleave

Se disparan cuando el cursor entra o sale de un elemento. Útiles para mostrar ayuda contextual o efectos visuales.

Ejemplo

Utilidades típicas
  • Mostrar tooltips (ayuda) cuando el usuario pasa el mouse.
  • Resaltar tarjetas o botones para mejorar UX.
  • Mostrar un menú desplegable (aunque hoy se prefiere click en móviles).

Eventos: keydown, keyup y keypress

Se usan cuando el usuario interactúa con el teclado. En general: keydown ocurre al presionar, keyup al soltar. keypress es más antiguo y hoy se usa menos.

Ejemplo

Escribe en el input y mira el log y el contador:

keydown: 0 keyup: 0 keypress: 0
Consejo: prueba Enter y mira qué evento se registra.
Utilidades típicas
  • Atajos: Ctrl + S para guardar.
  • Validación en tiempo real (ej: email).
  • Buscar mientras escribes (autocomplete).
Nota: keypress está “de salida” en muchas guías modernas; para la mayoría de casos usa keydown / keyup.

Evento: load

Ocurre cuando un recurso termina de cargar (ej: una imagen). Útil si quieres mostrar un “cargando…” hasta que esté lista.

Ejemplo

Utilidades típicas
  • Ocultar un spinner cuando una imagen/video termina de cargar.
  • Calcular tamaños reales después de cargar (ej: galerías).
  • Mostrar “Listo para imprimir” cuando recursos están listos.
Nota: en jQuery también existe $(document).ready() para correr código cuando el DOM está listo. Eso NO es “load de imagen”, es “DOM listo”.
